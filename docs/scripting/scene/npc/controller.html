🎭 Контроллер анимаций персонажа

Контроллер анимаций — это мощный инструмент управления анимациями 3D-моделей. Он позволяет реализовать сложные анимационные системы с поддержкой:

  • Состояний и переходов между ними
  • Смешивания анимаций (Blend Trees)
  • Процедурных анимаций
  • Масок костей
  • Поддержки Molang-выражений

🧩 Основные компоненты​

КомпонентОписание
ControllerГлавный класс, управляющий слоями анимаций.
LayerОтдельный слой анимации (приоритет, маска, смешивание).
StateСостояние: содержит Clip, BlendTree или ProceduralNode.
TransitionПереход между состояниями с условиями.
BlendTreeПлавное смешивание нескольких клипов по параметру.
ProceduralNodeУзел для скриптовой анимации на Molang.
ClipNodeУзел с обычной анимацией.
MaskМаска костей, определяющая влияние слоя.

📘 Molang в контроллере​

Molang — язык выражений, используемый для условий переходов, скоростей анимаций и т.п.

В HollowCore он работает через EntityQuery, предоставляющий переменные состояния сущности.

🔣 Примеры выражений​

  • q.is_moving — движется ли сущность
  • !q.is_alive — мертва ли сущность
  • min(q.ground_speed / 2, 3f) — ограничение скорости

📊 Поддерживаемые переменные​

ПеременнаяОписание
q.ground_speedСкорость по земле
q.is_movingДвижется ли
q.is_sneakingПриседает ли
q.is_sprintingБежит ли
q.is_jumpingПрыгает ли
q.velocity_yВертикальная скорость
q.is_flyingЛетит ли
q.fall_ticksСколько падает
q.is_swimmingПлавает ли
q.is_sittingСидит ли (в лодке и т.п.)
q.is_sleepingСпит ли
q.is_hurtПолучил ли урон
q.is_swingingАтакует ли
q.is_aliveЖив ли
q.head_rotПоворот головы

⚙️ Пример создания контроллера​

npc.controller = animationController {
    layer("myLayer", priority = 0, weight = 1f, mask = Mask.of("bone1", "!bone2"), blendMode = Additive) {
        stateMachine {
            state("idle") {
                clip("idleAnim", WrapMode.Loop)
            }
            state("walk") {
                clip("walkAnim", WrapMode.Loop)
            }
            transition("idle", "walk") {
                condition("query.is_walking")
                duration(0.5f)
            }
            transition("walk", "idle") {
                condition("!query.is_walking")
                duration(0.5f)
            }
        }
    }
}

✅ Объяснение:

  1. Создается слой с маской костей и режимом Additive
  2. Определены состояния idle и walk
  3. Переходы активируются через Molang-выражения

🧱 Слои анимаций​

Слои позволяют управлять приоритетом и избирательностью анимаций:

  • Имя — уникальный ID слоя

  • Приоритет — чем выше, тем позже применяется

  • Вес (weight) — степень влияния слоя (0.0 – 1.0)

  • Маска — кости, на которые влияет слой

  • Режим:

    • Override — замена предыдущих анимаций
    • Additive — добавление к текущей позе

🔁 Состояния и переходы​

Состояния​

state("run") {
    clip("runAnim", WrapMode.Loop)
}

Типы:

  • clip() — обычная анимация
  • blendTree {} — плавное смешивание
  • procedural {} — процедурная анимация

Переходы​

transition("idle", "run") {
    condition("query.is_running")
    duration(0.3f)
}

Параметры:

  • condition(...) — Molang-выражение
  • duration(...) — длительность перехода
  • exitTime(...) — минимальное время в текущем состоянии
  • clip(...) — необязательный переходный клип

🌈 Дерево смешивания​

Используется для плавного перехода между клипами на основе переменной:

blendTree {
    clip("walkSlow", threshold = 0f, WrapMode.Loop)
    clip("walkFast", threshold = 1f, WrapMode.Loop)
    factor("query.ground_speed", smoothingTime = 0.1f)
}
  • threshold — порог для клипа
  • factor(...) — переменная для интерполяции
  • smoothingTime — сглаживание перехода

🧬 Процедурные анимации​

Позволяют на лету изменять кости через Molang:

procedural {
    onEvaluate { context ->
        context.setBoneRotation("head", "query.head_rot")
        context.setBoneTranslation("arm", "vec3(query.arm_x, query.arm_y, query.arm_z)")
    }
}

🔁 Режимы проигрывания (WrapMode)​

РежимПоведение
OnceПроигрывается один раз
LoopЗациклен
PingPongВперёд-назад
ClampForeverЗамирает на последнем кадре

🪶 Маски​

Позволяют ограничить влияние анимаций:

Mask.of("spine", "!head", "left_arm")
  • Включает spine, left_arm
  • Исключает head

🔄 Обновление и отправка​

Контроллер:

  • Обновляется каждый кадр (update)
  • Применяет текущие состояния и переходы
  • Отправляется автоматически с сервера на клиент
  • Molang-компиляция выполняется асинхронно (~300мс)

🔔 Система оптимизирована и не вызывает зависаний.

Маски

💡 Используйте маски, чтобы изолировать движения частей тела, и BlendTree для плавных переходов!
